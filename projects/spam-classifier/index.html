Spam Detection System

Learning how machines understand text — not by magic, but by careful design.

After working on recommendation systems, a new question started bothering me: How does raw text actually become something a machine can reason about?

This curiosity led me to my third machine learning project — a Spam Detection System for Email and SMS messages. The goal wasn’t just classification, but understanding how text preprocessing, feature extraction, and model choice directly influence predictions.

End-to-end spam detection pipeline — from raw text input to prediction.

Building the NLP Pipeline

I started from raw, unstructured messages — messy, noisy, and inconsistent — and built a complete NLP pipeline from scratch. This included:

  • Text cleaning and normalization
  • Tokenization and stopword removal
  • Stemming to reduce vocabulary size
  • Transforming text into numerical form using TF-IDF

This phase reinforced an important lesson: in NLP, preprocessing often matters more than the model itself.

Model Exploration and Decision-Making

Instead of locking myself into a single algorithm, I experimented with multiple models, including Naive Bayes, Logistic Regression, SVM, Random Forest, and XGBoost.

What stood out quickly was that accuracy alone was misleading — especially for spam detection, where false positives matter more than raw correctness.

Based on precision and real-world relevance, I finalized TF-IDF with Multinomial Naive Bayes as the most reliable model. This decision wasn’t theoretical — it came from observing how different models behaved on imbalanced text data.

From Model to Usable System

To complete the system end-to-end, I deployed the trained model using a Streamlit web application, allowing users to input any message and instantly see whether it’s classified as spam or not.

This reinforced a simple but important principle for me: a model isn’t useful until someone can interact with it.

What This Project Taught Me

  • How machines interpret language through vectorization
  • Why preprocessing decisions compound downstream performance
  • How to compare models beyond surface-level metrics
  • The complete lifecycle of an NLP project — from raw text to deployment

This project marked a shift in how I approach machine learning. I stopped treating it as “apply an algorithm” and started seeing it as designing a pipeline where every decision compounds.
