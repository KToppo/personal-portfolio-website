Recommendation Systems

Learning through progressive design — understanding recommendations step by step, not by jumping straight to complexity.

My journey into recommendation systems started with a simple question: How do platforms suggest relevant content when they know very little about the user? Instead of jumping straight into complex models, I approached this domain step by step, building two projects that intentionally cover different recommendation strategies.

Progressive recommendation system design — from content-based filtering to collaborative personalization.

Phase 1: Content-Based Movie Recommendation System

My first project focused on content-based filtering, where recommendations are generated purely from item attributes rather than user behavior.

I built a movie recommender that suggests similar movies based on metadata such as genres, keywords, cast, and plot descriptions. The goal was to understand how feature representation and similarity measures drive recommendations without relying on user history.

Key learning outcomes from this phase:

  • Transforming raw textual metadata into numerical vectors
  • Why cosine similarity works well for high-dimensional text features
  • The importance of precomputing similarity matrices for real-time performance
  • Clear separation between offline training and online inference

To make the system interactive, I deployed it as a Streamlit web application and integrated the TMDB API to fetch movie posters dynamically. This phase highlighted the strengths and limitations of content-based systems — particularly their inability to adapt to evolving user preferences.

Phase 2: Book Recommendation System — Expanding Beyond Content

After completing the movie recommender, I realized an important gap: not all recommendation systems rely on content alone.

To address this, I built a Book Recommendation System that deliberately introduced two new paradigms:

  • Popularity-Based Recommendation
  • Collaborative Filtering

The popularity-based module ranks books using engagement thresholds and average ratings, mirroring real-world “Trending” or “Top Picks” systems where social proof matters.

The collaborative filtering module:

  • Builds a user–book interaction matrix
  • Measures similarity between books using shared user ratings
  • Recommends books liked by users with similar preferences

This project helped me understand the trade-offs between popularity, personalization, and cold-start problems, and how the same similarity logic behaves differently when driven by users instead of content.

What These Projects Represent Together

  • A structured understanding of multiple recommendation paradigms
  • Awareness of when each approach is appropriate
  • Practical system design over metric chasing
  • Progression from item-based logic to user-driven personalization

These projects were built while learning, guided by educational resources, but every implementation choice was made to understand the “why” behind recommendation systems, not just the “how”.

These are not production-scale recommenders — and that’s intentional. Their value lies in concept coverage and progression, not novelty. Presented honestly, they strengthen a profile. Oversold, they weaken it.
